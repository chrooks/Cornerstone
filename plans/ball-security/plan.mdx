## What this delivers

Cornerstone rates NBA players on a fixed 21-skill taxonomy, and a cohesion engine scores five-man lineups from those Skill Profiles. The engine's **Ball Security** trait (how well a lineup avoids turnovers) currently has no real skill behind it — it is approximated from `passer`, `pnr_ball_handler`, and `driver`. A turnover-prone star passer reads as "secure"; a careful low-assist scorer reads as careless.

After this work, a 22nd skill **`secure_handler`** ("Secure Handler") exists, derived directly from turnover statistics already in the stats blob. An admin can tune its thresholds against a draft Snapshot, rerun the pipeline for just that skill, review player-level tier diffs, and publish a Snapshot Release. The Ball Security trait reads the new skill, falling back to the legacy proxy only for Legends that have not been backfilled yet.

**Done means:** a published Snapshot Release whose profiles include `secure_handler`, a published Evaluation Version whose `ball_security` formula reads it, and the test suite proving the mapping and the fallback.

**Not in scope:** bulk Legends backfill (deferred to a follow-up issue — the fallback covers the window), synergy rules or accentuation entries for the new skill, and any new NBA.com stat fetching (all inputs exist: `tov`, `fga`, `fta`, `ast`, `usage_rate`, `touches`).

All seven product decisions were settled up front and are recorded in `feature_requests/ball-security-throughline.md` (Decision Ledger), `docs/adr/0004-secure-handler-stat-mapping.md`, and `docs/research/secure-handler-evidence.md`. A few implementation questions remain — see Open Questions at the bottom.

### Settled decisions (do not reopen)

    ["Confidence bucket", "HIGH — Claude never called", "Turnover economy is fully stat-measured; Claude adds no signal"],
    ["Trait consumption", "Skill tier when present; explicit proxy fallback when key absent", "Legends don't crater mid-transition; proxy retirement becomes a data event"],

<Callout id="trap-callout" tone="risk">

**The one genuine trap — missing vs. rated-None.** `tier_value` returns 0.0 both for a missing skill key and a skill rated None, and `_with_default_skills` fills every missing key with None BEFORE formulas evaluate. So key-absence is erased by the time the formula engine runs. The fallback trigger must be captured upstream: record whether the raw incoming profile contains `secure_handler` and thread that presence set into formula evaluation. A pipeline-rated None (careless player) must NOT inherit the proxy; an unbackfilled Legend must. Milestone 3 carries a targeted three-case test, and Milestone 1 verifies the pipeline writes skill keys even at None tier (if it omits them, that write path gets a one-line fix).

</Callout>

### Milestones

**M1 — The 22nd skill exists everywhere the taxonomy is named.** Add `secure_handler` to `HIGH_CONFIDENCE_SKILLS` and `SKILL_DEFINITIONS` in `backend/services/skills.py`; everything backend-side derives from those (pipeline allowlist, calibration allowlist, compositing, legends prompt, Evaluation Version validator). Update `frontend/lib/skills.ts` per its own header checklist (SKILL_CATEGORIES, SKILL_LABELS, SKILL_ABBREV, SKILL_TYPE_PRIORITY, SKILL_GROUPS, PUBLIC_SKILL_CATEGORIES, SKILL_DESCRIPTIONS). Fix stale count literals: the test hardcoding `len(HIGH_CONFIDENCE_SKILLS) == 6`, the legends prompt's "all 20 skills" prose, and cosmetic "21 skills" copy in landing/FAQ/admin pages. Verify None-tier keys are written (trap above).

**M2 — The threshold rule exists as data.** New migration modeled on `20260407000001_add_driver_skill.sql`: full JSONB rule into `skill_thresholds` (mirroring into `draft_skill_thresholds` only if drafts don't auto-seed from production — verify first), plus `anchor_players` rows. `always_flag_for_review: true` for the first cycle — deliberately floods the review queue once so the review pass has material. New unit tests evaluate synthetic blobs against the rule.

**M3 — The formula engine learns fallback; the trait reads the skill.** Add an optional `fallback` block (`when_missing` + `factors`) to `formula_engine.py` — all-or-nothing on `when_missing`, no nesting. Mirror in the hardcoded path for parity. Extend validator L8 so fallback factors validate against ALL_SKILLS. Three-case test: rated-Elite uses the skill, rated-None stays 0.0, key-absent gets the proxy.

**M4 — A new Evaluation Version carries the rewired trait.** Draft from active version; add `secure_handler` to `taxonomy.skills`; replace the `ball_security` composite formula with primary factor `secure_handler` (weight 1.0 — tier values already span 0–10 and the trait's theoretical max is 10.0) plus the fallback block reproducing current proxy coefficients. Validate, publish, confirm distribution warm-up.

**M5 — The live admin loop (driven interactively).** Open a draft Snapshot: edit a secure_handler threshold (stages a `threshold_edit` run, production untouched), rerun the pipeline with `skill_filter: ["secure_handler"]`, review player-level diffs, flags, and outliers against the research bands, adjusting and rerunning until the anchors land right, then publish the Snapshot Release. File the Legends-backfill follow-up issue.
